Unity 6 | Endless Runner Mechanics

Surprise Sprint

Overview: 2D endless runner with procedurally generated platforms, power-up systems, responsive controls, and persistent progression tracking.

Play Game

Technical Breakdown

Core Gameplay Loop

  • Continuous forward movement with input-based jump mechanics
  • Dash ability with cooldown management and coroutine handling
  • Collision-based game over detection and state management
  • Score tracking and high-score persistence via PlayerPrefs

Event Bus Architecture

  • Central Actions.cs with static event channels (OnGravity, OnPickup, OnTimer, OnGameOver)
  • Decoupled gameplay-UI communication through event subscription
  • Reduced inter-system coupling for maintainable codebase
  • State changes broadcast to all interested systems simultaneously

Procedural Level Generation

  • GroundSpawner with randomized platform X/Y gap variation
  • Object pooling via ObjectPooler for performance optimization
  • GroundDestroyer for cleanup when segments pass threshold
  • Infinite procedural generation for endless runner experience

Power-Up System

  • Trigger-based random effect selection (size buff, debuff, reverse gravity)
  • Timed lifecycle management with countdown tracking
  • SFX triggers and visual feedback via PowerUpUI
  • Event emission for cross-system power-up awareness

UI & Menu Systems

  • Main menu flow with scene management
  • Pause/settings menu with runtime configuration
  • Resolution, fullscreen, and graphics quality controls
  • Volume persistence through PlayerPrefs + AudioMixer integration

Audio & Visual Integration

  • Background audio pause/resume on game state changes
  • Audio stop on game over with proper cleanup
  • CameraMovement following player delta position
  • PostProcessingEffects with vignette toggling for gravity state

Description

Surprise Sprint is a 2D endless runner built in Unity 6 focusing on responsive, replayable gameplay with lightweight architecture. The player continuously moves forward, jumping across randomized platforms while managing a dash ability and interacting with timed power-ups. The game features a main menu, in-game pause/settings UI with audio and graphics controls, score tracking, and persistent high-score storage.

What I Have Done

  • Implemented core player gameplay loop with forward running, jump mechanics, and dash cooldown system
  • Built procedural level generation with randomized platform spawning and object pooling for performance
  • Created power-up system with buff/debuff/gravity variations including duration management and audio-visual feedback
  • Developed game state UI systems for score, high score, power-up labels, and effect countdown timers
  • Built complete menu and settings systems with graphics, resolution, and fullscreen options
  • Implemented event-driven architecture using central Actions.cs for decoupled system communication
  • Integrated camera follow logic and contextual audio control across pause/game-over states
  • Implemented persistent high-score and volume settings via PlayerPrefs with AudioMixer integration
